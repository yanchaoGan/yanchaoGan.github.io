restful web api 设计思想

主要阅读了 ( 1 , 2, 3)

一些感想

  1. API域名与网站域名分开，比如使用 https://api.example.com 这样的方式

  2. API请求中应该加入API版本号，比如在URL中加入版本号，如 https://api.example.com/v1/， 这样当API升级的或者改动的时候，可以保留旧的API服务器，把新的API版本号 添加到 到https://api.example.com/v2/ 上，这样使用旧的API的App。这点尤为有效。

  3. 使用http method 对应 server常用 的CRUD (Create, Read, Update, Delete)操作，就像是 定位到 数据库的一张table，来直接进行操作。清晰简洁。
